# 试题列表页面刷新状态恢复计划

## 问题分析
刷新网页时，试题列表收起恢复默认状态。原因是 `currentViewingSubject` 变量存储在内存中，刷新后丢失。

## 解决方案
使用 URL hash 来保存当前正在查看的科目信息，刷新后根据 hash 恢复状态。同时使用 localStorage 保存滚动位置。

## 具体修改

### 1. admin.js 修改

#### 1.1 修改 `loadSubjectQuestions` 函数
- 添加 URL hash 来记录正在查看的科目，如 `#view/subjectId/subjectName`
- 示例：`#view/1/科目名称`

#### 1.2 添加滚动位置保存
- 监听 questionListArea 容器的 scroll 事件
- 保存滚动百分比到 localStorage，key 为 `questionListScroll_{subjectId}`

#### 1.3 添加 hash 恢复逻辑
- 在页面加载时检查 hash 格式是否为 `#view/xxx/xxx`
- 如果是，则自动调用 `loadSubjectQuestions` 恢复试题列表
- 恢复后根据保存的滚动百分比恢复滚动位置

#### 1.4 修改 `backToAdmin` 函数
- 清除对应的 hash

### 2. main.js 修改

#### 2.1 修改 `handleHashChange` 函数
- 识别 `#view/` 开头的 hash，不执行普通的 tab 切换逻辑
- 当检测到 `#view/` hash 时，解析出科目信息并调用 `loadSubjectQuestions`

### 3. 滚动位置恢复时机
- 试题列表渲染完成后，使用 `setTimeout` 等待 DOM 更新
- 根据保存的滚动百分比计算实际像素位置并滚动
- 滚动位置 key 格式：`questionListScroll_{subjectId}`

## 预期效果
- 用户在试题列表页面刷新网页后，页面会自动根据 hash 恢复显示该科目的试题列表
- 滚动位置也会恢复到之前的百分比位置（约50%处）
- 点击"返回"按钮时，hash 被清除，列表收起恢复到 admin 面板
