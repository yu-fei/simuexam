# 试题列表分页功能优化计划

## 背景
用户反馈：在管理员界面点击"查看试题列表"时显示的界面，当前已实现前端分页（每页50题），但后端一次性返回所有题目。

## 当前状况

### 前端分页（已实现）
- 文件：`templates/js/admin.js`
- 每页显示 50 题
- 完整的分页 UI：上一页/下一页、页码跳转、省略号显示

### 后端（未分页）
- 文件：`routes/question.py` - API `/api/subjects/<int:subject_id>/questions`
- 一次性返回指定科目的**所有**题目
- 文件：`services/question_service.py` - 查询使用 `fetch_all_questions()`

## 潜在问题
当题目数量很多时，后端一次性返回所有数据会导致：
1. API 响应慢
2. 前端内存占用高
3. 网络传输量大

## 优化方案：后端加分页

### 步骤 1：修改后端服务层
- 文件：`services/question_service.py`
- 修改 `fetch_questions_by_subject` 添加分页支持（limit/offset）

### 步骤 2：修改后端路由
- 文件：`routes/question.py`
- 添加 `page` 和 `page_size` 查询参数
- 返回分页元数据：`{questions: [], total: N, page: N, page_size: N, total_pages: N}`

### 步骤 3：修改前端 API 封装
- 文件：`templates/js/api.js`
- `getQuestionsBySubject` 添加分页参数支持

### 步骤 4：修改前端加载逻辑
- 文件：`templates/js/admin.js`
- 改为按需加载：只请求当前页数据
- 使用后端返回的分页信息更新分页状态

## 预估工作量
- 后端修改：约 20-30 行代码
- 前端修改：约 30-40 行代码
- 测试验证：手动测试几个科目
