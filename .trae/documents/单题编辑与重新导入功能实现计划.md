# 单题编辑与重新导入功能实现计划

## 需求背景

当批量导入的试题中某些题目格式有问题时，需要能针对单题进行编辑或重新导入。

### 额外需求：考试过程中处理问题题目

* 在考试过程中发现了错题（格式问题），想立即删除或重新导入

* 需要处理好考试session状态

### 额外需求：页面刷新后保持状态

* 当前页面通过JS控制显示/隐藏，刷新后状态丢失

* 解决方案：使用URL hash（如 `#exam`, `#history`, `#admin`）保存当前tab状态

* 页面加载时读取URL hash来决定显示哪个tab

* **考试模式特殊处理**：
  - 如果 hash 是 `#exam`，自动检查后端是否有进行中的考试
  - 如果有，自动恢复考试状态（不弹窗）
  - 如果没有，返回首页并显示"没有进行中的考试"

* **新打开标签页**：
  - 默认 hash 为空或 `#exam`，自动检查进行中的考试
  - 如果有进行中的考试，自动进入考试恢复流程
  - 用户也可以从历史记录中点继续考试

## 数据关系分析

### 现有数据表关系

```
exam_sessions (考试会话)
├── question_ids: TEXT (JSON数组，存储考试时的题目ID列表)
└── exam_session_details (答题详情)
    └── question_id → questions.id (ON DELETE CASCADE)
```

### 删除单题时的处理问题

1. **exam\_sessions.question\_ids** - 存储的是JSON数组，被删除的试题ID会仍然保留在历史记录中
2. **exam\_session\_details** - 有 ON DELETE CASCADE 外键，物理删除题目时会自动删除相关答题记录
3. **考试中的题目** - 如果题目正在 in\_progress 的考试中使用，需要处理session状态

### 解决方案：彻底删除 + 清理历史记录

1. **物理删除题目** - 直接从 questions 表删除
2. **清理 exam\_session\_details** - ON DELETE CASCADE 会自动处理
3. **更新 exam\_sessions.question\_ids** - 从JSON数组中移除被删除的题目ID
4. **更新 exam\_sessions.total\_questions** - 相应减少总题数
5. **处理进行中的考试** - 如果题目正在 in\_progress 考试中使用：

   * 从该考试的 question\_ids 中移除

   * 减少 total\_questions

   * 如果用户已经答过该题，需要同时清理 exam\_session\_details 中的答题记录

### 删除单题的具体操作步骤

```python
# 1. 开启事务
# 2. 查找所有使用该题目的 exam_sessions
# 3. 对于进行中的考试(in_progress)：
#    - 从 question_ids 中移除题目ID
#    - 减少 total_questions
#    - 清理 exam_session_details 中已答记录（如果有）
# 4. 对于已完成的考试：
#    - 从 question_ids 中移除题目ID
#    - 减少 total_questions（可选，历史记录可能不需要改）
#    - exam_session_details 会被 CASCADE 自动删除
# 5. 物理删除 questions 表中的记录
# 6. 提交事务
```

## 实现步骤

### 第一步：后端 - 新增API接口

#### 1.1 获取科目试题列表

* 接口：`GET /api/subjects/<subject_id>/questions`

* 功能：返回指定科目的所有试题

#### 1.2 获取单题详情

* 接口：`GET /api/questions/<question_id>`

* 功能：返回指定试题的完整信息

#### 1.3 更新单题

* 接口：`PUT /api/questions/<question_id>`

* 功能：更新试题的内容、类型、选项、正确答案、解析

#### 1.4 新增单题

* 接口：`POST /api/questions`

* 功能：在指定科目下新增一道试题

#### 1.5 删除单题（彻底删除 + 清理历史记录）

* 接口：`DELETE /api/questions/<question_id>`

* 功能：

  1. 物理删除 questions 表中的记录
  2. exam\_session\_details 会被 CASCADE 自动删除
  3. 从所有 exam\_sessions 的 question\_ids JSON数组中移除该题目ID
  4. 更新对应 exam\_sessions 的 total\_questions 字段

#### 1.6 优化试题解析函数
- 复用 `parse_uploaded_txt` 的解析逻辑并优化
- 支持解析单题txt格式（与批量导入格式相同）
- 支持题干换行（多行题干合并为一道题）
- 支持中文逗号选项（`A、xxx`、`A. xxx`、`A，xxx`）

## TXT试题格式说明

### 当前支持的格式
```
判断题格式：
正确 这是一道判断题
错误 这是一道判断题
对 这是一道判断题
错 这是一道判断题
✓ 这是一道判断题
√ 这是一道判断题
是 这是一道判断题
否 这是一道判断题
T/True 这是一道判断题
F/False 这是一道判断题

单选题格式：
A 这是一道单选题?
A、选项1
B、选项2
C、选项3
D、选项4

多选题格式：
AB 这是一道多选题?
A，选项1
B. 选项2
C、选项3
D. 选项4
```

### 需要优化的点
1. **支持题干换行** - 有些题目的题干比较长，会分成多行
2. **支持中文逗号选项** - 选项格式应该支持 `A、xxx`、`A. xxx`、`A，xxx` 三种格式
